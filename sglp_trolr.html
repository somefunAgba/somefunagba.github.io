Trust-Region Optimal Learning Rates

A Modular Principle

Recall that momentum (Polyak, 1969) is a lowpass regularized stochastic gradient algorithm. One that applies a first-order lowpass filter to its stochastic gradient, before multiplying by a possibly optimal iteration-dependent learning rate. Here, more specifically, we show how using lowpass regularization, Adaptive Moment Estimation (Adam) and Momentum Orthogonalized with Newton-Schultz (Muon) encode the principle of trust-region optimal learning rates for the lowpass regularized stochastic gradient algorithm.

So What?: Modular Design and Trust Regions

An additional insight from Momentum as Lowpass Filter Design is that stochastic gradient algorithms can be designed to implement a modular strategy consisting of two principles:

  1. Gradient Filtering via a Lopass Filter: Designing \(\beta, \gamma\).
  2. Gradient Normalization via the Learning-rate: Scaling by a Data-driven substitute for \(1/\lambda\).

We will now show that popular modern deep learning optimizers like Adam and Muon implement these two principles.

The Learning-Rate Subsystem: Trust-Region Normalization Principle

The second module of the framework is the learning-rate subsystem, which is strategically separated from the temporal smoothing of the gradient filter.

We define the general update rule:

\[w(t+1) = w[t] - \alpha[t] v[t]\]

With normalization:

\[\alpha[t] = \mu \lambda^{-1}[t]\]

Where:

  • \(\mu\) = Trust-Region Radius
  • \(\lambda[t]\) = gradient scale

Modular Design Principle

  1. Filter \(\rv[t] = \sH_{\beta,\gamma}\{\rg[t]\}\)

  2. Normalize \(\rw(t+1) = \rw[t] - \mu \lambda^{-1}[t]\,\rv[t]\)

    1. Filter Design

    Choose \(\beta, \gamma\) to shape the gradient signal:

\[v[t] = \beta v(t-1) + \gamma g[t]\]

2. Learning-Rate Normalization

Apply a trust-region scaling:

\[\text{update} = \mu \cdot \lambda^{-1} \cdot v[t]\]

where:

  • \(\lambda\) = scale of the gradient
  • \(\mu\) = step-size budget

Normalizing by \(1/\lambda\) defines a trust-region radius \(\mu\):

“Never take a step larger than this safe boundary.”

Adam (Vectorized Formulation)

Performs coordinate-wise normalization:

\[\alpha(i) = \frac{\mu}{\sqrt{\mathbb{E}[g^2(i)]}}\] \[\lambda_i = \sqrt{\mathbb{E}[g_i^2]}\]

Update:

\[\Delta x_i = \mu \frac{v_i}{\lambda_i}\]
  • Normalizes each coordinate independently
  • Enforces:
\[\|\Delta x_i\|^2 \le \mu^2\]

enforcing the trust-region constraint on the parameter update increment: \(\mathbb{E}[\Delta^2] \le \mu^2\)

For \(w[t] \in \mathbb{R}^n\):

\[b(t,i) = \rho b(t-1,i) + (1-\rho) g^2(t,i)\]

Constraint:

\[\mathbb{E}[\Delta^2(t+1,i)] \le \mu^2\] \[\mathbb{E}[\|\Delta(t+1)\|_2^2] \le n\mu^2\]

Muon (Matrix-valued Operator-level Formulation)

Uses matrix inverse square-root normalization: \(C^{-1/2}\)
ensuring: \(\|\Delta\|_F^2 \le \mu^2\)

Uses matrix scaling:

\[\Delta W = \mu \, (\mathbf{G}^\top \mathbf{G})^{-1/2} \, v\]
  • Typically approximated via Newton–Schulz iteration
  • Enforces Frobenius constraint:
\[\|\Delta W\|_F^2 \le \mu^2\]

Using Frobenius geometry:

\[\langle A, B \rangle_F = \mathrm{tr}(A^T B)\]

Scale:

\[C[t] = v[t]v[t]^T\]

Normalized gradient:

\[\bar{v}[t] = C^{-1/2}[t] v[t]\]

Constraint:

\[\|\Delta(t+1)\|_F^2 = n\mu^2\]

Implementation via Iterative Methods

Matrix inverse square root computed via Newton–Schulz iteration (Lakić polynomial family).

Unified Insight

Both follow a general update form:

\[\Delta[t+1] = \gain{\mu} \cdot \lambda^{-1}[t] \cdot \rv[t]\]

The only difference is how \(\lambda\) is defined.

New optimizers can be designed explicitly by (i) designing a gradient filter and (ii) choosing a learning-rate rule that encodes gradient normalization.

Conclusion: Your New Perspective on Learning and Optimizers

Instead of relying on physical metaphors, we can:

  • Treat gradients as signals
  • Design filters explicitly
  • Control transient and steady-state behavior via the trust-region

You have likely moved beyond seeing AI training as a black box of weights and biases. You now understand it as a sophisticated signal processing task. Momentum is the heartbeat of this process, a digital filter that separates the signal of the “truth” from the noise of the data.

3-Point Takeaway

  • Momentum is a Filter
    It is a first-order lowpass filter designed to attenuate high-frequency noise and preserve the underlying signal.

  • Poles and Zeros are the Keys
    The difference between Heavy-Ball and Nesterov is determined by: \(\gamma\)

  • Stability is the True Goal
    Momentum expands the stability region, allowing aggressive learning rates while maintaining safe updates within a trust region: \(\mu\) We will show in a follow-up work that the optimal $\mu$ changes as a result of the kurtosis $\kappa$, tail-heaviness of the noise distribution.

Open Question

If popular algorithms encode this modular principle, then: What unexplored filtering plus trust-region design principles could unlock the next generation of optimizers?
